CS 338 Notes

Defintitions

Attribute Table column OR a given property of a tuple.
Database (DB) A collection of files (records and dictionaries) containing data
Database management system (DBMS) Collection of programs that manipulate databases:
  • Maintains, replicates, recovers data
  • Applies access restrictions
  • Ensures data independence (physically / logically)
  • Ensures data concurrency (many things can be done to a DB at any given time at multiple locations)
  • Ensures data consistency, i.e. byte order
  • Enforces data integrity, i.e. the data means something
Data definition language (DDL) Specifies database schemas. Examples: ALTER TABLE, TRUNCATE, DROP.
Data manipulation language (DML) Manipulates data. Examples: SELECT, UPDATE, INSERT, DELETE.
Database catalogue Data dictionary; data about the data.
Domain Set of values and functions defined over them. An ADT.
Domain constraint If attribute specifies an int, you must supply an int.
Foreign key Attribute that is the primary key of another relation.
Integrity
  • Entity integrity: primary key cannot be null in value.
  • Referential integrity: value of a foreign key must be found in its source table. For example, if you use user = 111 in transactions, you must have user 111 in your users table.
Null not known; does not apply.
Relation Table.
Relation Schema Names of all attributes in the relation.
Security Could be physical (theft, fire, ...), authorisation (user/pass) and authentication (user level).
Tuple Table row.
